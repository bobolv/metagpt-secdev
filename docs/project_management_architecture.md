# 项目管理与生命周期模板

这个仓库当前已经具备两条线：

1. MetaGPT 原生的“软件公司”流程，核心在 `Action -> Role -> Team -> Environment`。
2. 这个 fork 里新增的项目生命周期文档原型，核心在 `tools/lifecycle_orchestrator.py` 和 `tools/lifecycle_agent.py`。

为了让后续 Web 页面、项目台账和阶段文档生成走同一套配置，新增了 `metagpt/project_management/registry.py` 作为统一模板层。

## 角色选择

Web 页面里支持的角色应该直接来自模板注册表，而不是散落在入口脚本里硬编码。

当前建议的角色目录：

- 团队负责人
- 项目经理
- 产品经理
- 架构师
- 开发经理
- 开发工程师
- 测试经理
- 研究员
- 售后服务
- 销售

每个角色都保留两个信息：

- 展示名，供页面和表单使用
- MetaGPT 角色类路径，供后端真正实例化执行

## 生命周期阶段

建议的生命周期阶段：

- 立项阶段
- 需求调研阶段
- 设计阶段
- 开发阶段
- 测试阶段
- 部署阶段
- 试运行阶段
- 验收阶段

每个阶段都拆成两层：

- 阶段文档清单：一次性产物，比如需求分析、系统设计、测试报告
- 定期文档：周报、月报这类周期产物

## 推荐分配

建议的默认分工是：

- 立项与收口：团队负责人、项目经理
- 需求与调研：产品经理、研究员、销售
- 设计：架构师、项目经理、开发经理
- 开发：开发经理、开发工程师
- 测试：测试经理、开发工程师
- 部署与试运行：开发经理、测试经理、售后服务
- 验收：项目经理、团队负责人、产品经理

## 后续迭代建议

下一步可以把这层模板接到三类能力上：

1. Web 项目管理页面
2. 后端项目台账 API
3. 定时生成周报、月报和阶段总结的任务调度

这样就能从“脚本生成文档”升级成“项目驱动的协作系统”。
